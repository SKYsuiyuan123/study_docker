
## 微服务的优缺点

#### 优点

1. 去中心化，所有功能全部变为服务，彼此服务和功能都是独立的，互相依赖的关系。每个服务只负责自己的事情，单一职责原则。

2. 可以使用不同的语言开发，只需要暴漏出接口。

3. 单一服务的团队规模可以变成 2-5 个人，单一部署。

4. 对于刚入职的员工非常友好

#### 缺点

1. 服务器成本高。
2. 对架构要求提高。
3. 运维、测试成本提高。
  
因此需要使用 Docker 解决微服务的 服务器众多成本过高的问题。


## Docker CI, CD

CI (continuous integration) 持续集成：编写代码时，完成了一个功能后，立即提交代码到 GIT 仓库中，将项目重新的构建并测试效果。
  - 可以快速发现错误。
  - 防止代码偏离主分支。

CD () 持续交付，持续部署。
  - 持续交付：将代码交付给专业的测试团队去测试。
  - 持续部署：将测试通过的代码，发布到更高环境中。(生产)
